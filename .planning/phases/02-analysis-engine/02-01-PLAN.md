# Phase 2 Plan 01: Database Vectors and Text Chunking

## Objective
- **What:** Add `pgvector` support to the database schema and build a utility to chunk transcripts.
- **Why:** We need to store semantic embeddings for transcript sections before we can score them alignment.
- **Output:** Updated models in `src/db/models.py`, new `TranscriptChunk` table, and a new `src/analysis/chunker.py` module.
